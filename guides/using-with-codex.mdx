---
title: "Use SKILL.md with Codex for extracted design tokens"
description: "Install a SKILL.md file into your project's .agents/skills/ directory so Codex can apply extracted design tokens and component rules during generation."
---

Codex reads skill files from the `.agents/skills/` directory at your project root. Installing a SKILL.md generated by the DESIGN.md Style Extractor extension into this location gives Codex access to the full set of extracted design tokens — typography scale, color palette, spacing values, radius and shadow tokens — along with component rules, accessibility requirements, and brand context captured from a live site.

## How Codex discovers skills

Codex looks for skill files under `.agents/skills/` when handling generation tasks. The `name` field in each skill's YAML frontmatter identifies the skill and is used to activate it. For a skill generated from a site called Vercel, for example, the frontmatter would contain `name: design-system-vercel`. This name is visible at the top of the generated SKILL.md file.

## Quick install

Quick Install uses the browser's File System Access API to write the file directly into your project without a manual copy step.

<Steps>
  <Step title="Navigate to the target site">
    Open the website whose design system you want to capture. Use the deployed production version of the site to get the most accurate computed styles.
  </Step>
  <Step title="Open the extension and select SKILL.md mode">
    Click the extension icon in Chrome. In the mode switcher at the top of the popup, select **SKILL.md**. The extension extracts styles from the active tab and generates the skill content.
  </Step>
  <Step title="Click the Codex button in Quick Install">
    In the **Quick install** section at the bottom of the popup, click the **Codex** button. This calls `installQuick("codex")` internally, which resolves the target directory as `.agents/skills/design-system`.
  </Step>
  <Step title="Select your project root folder">
    A directory picker dialog will open. Select the root folder of your project — the folder that contains your source files and where `.agents/` will be created. The extension requests `readwrite` access to create subdirectories and write the file.
  </Step>
  <Step title="Confirm installation">
    The extension walks the path `.agents` → `skills` → `design-system`, creating each directory if it does not exist, then writes `SKILL.md`. A confirmation message appears in the popup: **"Installed for Codex at .agents/skills/design-system/"**. The installed path relative to your project root is:

    ```text
    .agents/skills/design-system/SKILL.md
    ```
  </Step>
</Steps>

## Manual installation

If the directory picker is unavailable — for example, in browsers that do not support the File System Access API — you can download and place the file manually.

<Steps>
  <Step title="Generate and download SKILL.md">
    With SKILL.md mode active, click the download button to save `SKILL.md` to your Downloads folder. Alternatively, click the copy button to copy the full content to your clipboard.
  </Step>
  <Step title="Create the skills directory">
    In your project root, create the required directory structure:

    ```bash
    mkdir -p .agents/skills/design-system
    ```
  </Step>
  <Step title="Place SKILL.md in the directory">
    Move the downloaded file into the new directory. The final path relative to your project root should be:

    ```text
    .agents/skills/design-system/SKILL.md
    ```
  </Step>
</Steps>

## Activating the skill in Codex

Reference the skill by the `name` value from its frontmatter. If the generated file begins with `name: design-system-notion`, activate it in a Codex prompt like this:

```text
Use the design-system-notion skill to implement this data table component.
```

Codex will apply the token definitions, component state requirements, accessibility constraints (WCAG 2.2 AA target, keyboard-first interactions, focus-visible rules), and the writing tone specified in the skill when generating output.

## Using DESIGN.md as an alternative

If you want to provide design context without installing a persistent file, switch to **DESIGN.md** mode in the extension, copy the generated content, and paste it directly into your Codex conversation before describing the UI task. DESIGN.md omits the YAML frontmatter and managed section markers that SKILL.md uses, making it a lighter option for single-session context. This is useful when working across multiple projects or when you want to test a design system before committing it to your repository.

<Note>
  DESIGN.md content pasted into a conversation is not persisted between sessions. For ongoing projects, install the SKILL.md file so the skill is always available without re-pasting.
</Note>
