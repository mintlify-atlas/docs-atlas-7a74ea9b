The DESIGN.md Style Extractor is a Chrome extension that reads the computed styles of any live website — typography, colors, spacing, radius, shadows, and motion — and generates ready-to-use design system files. These files give AI coding tools the context they need to build UI that matches an existing design language.

The extension produces two output formats: **DESIGN.md** for human-readable design system documentation, and **SKILL.md** for agent-ready skill files you can drop directly into your Claude Code, Codex, or Cursor project.

## How it works

<Steps>
  <Step title="Install the extension">
    Load the unpacked extension in Chrome via `chrome://extensions`. Enable Developer mode, click **Load unpacked**, and select the project folder.
  </Step>
  <Step title="Navigate to any website">
    Open the tab you want to extract from. The extension reads computed styles from visible DOM elements — no special access or login is required.
  </Step>
  <Step title="Generate your file">
    Click the extension icon. Choose **DESIGN.md** or **SKILL.md** mode, then click **Refresh** to run extraction and generate the output.
  </Step>
  <Step title="Install to your AI tool">
    Use **Quick Install** to write the SKILL.md directly into your Claude Code, Codex, or Cursor project directory, or download and copy it manually.
  </Step>
</Steps>

## What gets extracted

The extension samples up to 280 visible elements and extracts six categories of design tokens from their computed styles:

| Category | What is captured |
|---|---|
| Typography | Font family, size scale, line height, weight, letter spacing |
| Colors | Text, background, border, and focus ring colors |
| Spacing | Margin and padding values across the scale |
| Radius | Border radius values |
| Shadows | Box shadow values |
| Motion | Transition and animation durations and easing functions |

<Tip>
  For best results, extract from a page that has multiple UI components visible — buttons, forms, navigation, cards. More visible elements means higher token coverage and more confident site profiling.
</Tip>
