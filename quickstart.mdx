title: "Quickstart: extract styles and generate your first file"
description: "Extract styles from any live website and generate a DESIGN.md or SKILL.md file using the DESIGN.md Style Extractor Chrome extension in five steps."
---

The fastest path to a design system file is to navigate to a site with rich visible UI, open the extension, and click **Refresh**. The extension samples up to 280 visible elements, normalizes the extracted tokens, and generates a complete markdown file in seconds.

<Note>
  Extraction does not work on `chrome://` pages or the Chrome Web Store. Navigate to any regular website before opening the extension.
</Note>

## Generate your first file

<Steps>
  <Step title="Install the extension">
    If you haven't already, load the extension in Chrome. Open `chrome://extensions`, enable **Developer mode**, click **Load unpacked**, and select the project folder. See [Installation](/installation) for detailed steps.
  </Step>
  <Step title="Navigate to a site with rich UI">
    Open any website you want to extract from. For best results, choose a page with visible buttons, forms, navigation, and cards — more visible elements means better token coverage.

    <Tip>
      Try a product's marketing or dashboard page rather than a plain landing page. The more UI components visible, the higher the token coverage and site profile confidence.
    </Tip>
  </Step>
  <Step title="Open the extension">
    Click the DESIGN.md Style Extractor icon in your Chrome toolbar. If you don't see it, click the puzzle icon and pin the extension.

    The popup opens with the mode switcher at the top (DESIGN.md / SKILL.md) and a markdown preview area below.
  </Step>
  <Step title="Choose your output mode">
    Select the mode that matches your use case:

    - **DESIGN.md** — human-readable design system documentation you can share or paste into an AI conversation
    - **SKILL.md** — agent-ready skill file with frontmatter for direct installation into Claude Code, Codex, or Cursor

    The extension remembers your last mode selection using `chrome.storage.local`.
  </Step>
  <Step title="Run extraction">
    Click the **Refresh** button (circular arrow icon). The extension injects the content script into the active tab, collects computed styles from up to 280 visible elements, normalizes the values into semantic tokens, and generates the output markdown.

    The preview area shows the generated file when complete. Any validation warnings appear in an issues list below the mode switcher.
  </Step>
  <Step title="Review the output">
    Read through the generated file in the preview. Check the **Help** panel (? button) to see:
    - How many elements were sampled
    - Token coverage breakdown (typography, color, spacing, radius, shadow, motion)
    - Site profiling evidence and confidence level
    - How many validation checks passed
  </Step>
  <Step title="Save or install the file">
    You have three options:

    **Download** — Click the download icon to save the file as `DESIGN.md` or `SKILL.md`.

    **Copy** — Click the copy icon to copy the markdown to your clipboard.

    **Quick Install** — Click a provider button to write the SKILL.md directly into your project directory.
  </Step>
</Steps>

## Quick Install target paths

When you use Quick Install, the extension writes `SKILL.md` to a subdirectory inside the project root you select:

| Provider | Target path |
|---|---|
| Claude Code | `.claude/skills/design-system/SKILL.md` |
| Codex | `.agents/skills/design-system/SKILL.md` |
| Cursor | `.cursor/skills/design-system/SKILL.md` |

The extension creates intermediate directories automatically using the File System Access API. If the API is not available, it falls back to copying to clipboard and downloading the file.

<Warning>
  When the directory picker opens, select your **project root** — not a subdirectory. The extension appends the full target path from the root.
</Warning>

## How extraction works

The extension's content script (`content-script.js`) uses a prioritized selector list to find visible elements:

```javascript
const selectors = [
  "body",
  "h1,h2,h3,h4,h5,h6",
  "p",
  "a",
  "button",
  "input,textarea,select",
  "label",
  "nav,header,footer,main,section,article,aside",
  "ul li,ol li",
  "table,th,td",
  "[role='button']",
  "[class*='card']",
  "[class*='btn']",
  "[tabindex]"
];
```

Elements are skipped if they have zero dimensions, `display: none`, or `visibility: hidden`. The script samples up to 280 unique visible elements and returns their computed typography, color, spacing, radius, shadow, and motion values.
